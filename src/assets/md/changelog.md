# 更新日志

本文件记录时迹各版本的主要变更内容，格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/) 规范。

---

## [1.1.0] - 2026-07-02

### 新增

- **媒体修复工具**：新增独立页面，可查找归档库中缺少 EXIF 拍摄时间的文件，并批量将归档时推算的时间写回文件元数据；支持关键词搜索、可调列宽与列显隐
- **重复查找重设计**：改为两阶段哈希策略（先按文件大小分组，再对同大小文件计算 SHA1），结果页新增可调列宽、列显隐及按扩展名 / 文件大小过滤
- **分类规则 / 命名规则用户隔离**：每个用户现在拥有独立的规则集，首次访问时自动初始化默认规则；所有用户均可自行管理自己的规则，不再仅限管理员
- **日志下载**：设置页新增应用日志（`archiver.log`，含日志轮转）与运行日志（`info.log`）的下载入口，仅管理员可用
- **移动端适配**：侧边栏改为抽屉式，操作控件在小屏幕下自动收拢
- **文件浏览**：归档模式选择器旁新增说明图标，区分手动归档与定时自动归档
- **操作日志**：页面顶部新增日志保留时间说明提醒

### 变更

- 重复查找改为纯内存方案，扫描结果不再写入任务表，彻底解决任务列表被污染的问题
- 运行日志改为直接读取 `archiver.log` 文件内容，不再查询数据库
- 文件浏览复制 / 移动模式选择器从筛选区移至操作行

### 修复

- 媒体修复工具：搜索框特殊字符（`%` `_` `\`）未转义导致查询结果异常
- 媒体修复工具：EXIF 写回成功后未更新数据库 `exif_date` 字段，筛选状态不准确
- 重复查找：删除文件后未同步清理内存扫描结果，统计数据滞后不准
- 文件浏览：年份筛选最多只显示到 2020 年（参数计算错误）
- 后端：日志下载接口流式响应触发 `ERR_INCOMPLETE_CHUNKED_ENCODING`
- 后端：SHA1 去重查询未带 `user_id` 过滤，可能误判跨用户重复文件
- 规则系统：升级后存量规则 `user_id=NULL`，定时任务在首次访问规则页前触发时以空规则集运行
- 会员开通时间显示为原始 UTC 纳秒字符串，未转换为本地时间

### 安全

- 重复查找的扫描与删除操作改用 `is_path_usable_by_user` 鉴权，防止跨用户访问他人个人空间
- 规则写操作全部加入 `AND user_id=?` 约束，防止跨用户越权修改

## [1.0.0] - 2026-06-23

> 首次正式发布。

### 新增

- 智能扫描：递归扫描用户指定目录下的全部文件，不限制文件类型
- 增量识别：基于文件大小与修改时间判断文件是否变化，未变化文件自动跳过，并标记源目录中已消失的文件
- 多源时间信息提取：依次尝试 EXIF/元数据拍摄时间、文件名时间戳、文件系统修改时间，确保归档时间尽可能准确
- 内容去重：扫描阶段基于 SHA1 哈希值自动识别重复文件，避免重复归档
- 自动归档：按 `{年}/{月}/{分类}/` 层级结构整理文件，结构统一且长期可维护
- 双归档模式：支持「复制」与「移动」两种模式，满足不同场景下的数据安全需求
- 自定义分类规则：支持按扩展名、文件名、来源文件夹、EXIF 品牌与型号等维度，结合优先级匹配分类
- 内置默认分类：未匹配自定义规则的文件按类型归入 Photos/Videos/Audio/Doc/Others
- 条件化自动命名：支持按前缀、后缀、来源文件夹、哈希特征、正则表达式等条件触发模板化重命名
- 文件浏览：支持按状态、类型、年月、关键词筛选，批量归档、批量删除、按状态清空
- 重复文件查找与清理：独立工具，基于 SHA1 哈希值对任意目录分组查找重复文件
- 定时任务调度：基于 Cron 表达式自动执行扫描与归档任务，支持逐任务单独配置
- 仪表盘统计：按类型、状态、年月等维度汇总文件处理情况
- 操作日志记录与查询：完整留存历史操作记录，便于追溯
- 多用户数据隔离：不同用户的任务、文件记录、定时任务、日志严格隔离，互不可见
- 数据库备份：管理员可导出/导入全机数据库，用于整机级别的备份与迁移
- 免费额度与终身会员授权：基于设备机器码离线校验，一次开通永久有效
- Web 管理界面：提供完整的任务配置与管理能力
