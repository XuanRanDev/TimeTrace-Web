# 使用教程

本指南将引导您完成时迹的安装、配置及基本操作流程。

---

## 一、安装应用

### 方式一：通过 fnOS 应用中心安装（推荐）

1. 打开 fnOS 系统，进入 **应用中心**
2. 在搜索栏中输入「**时迹**」
3. 找到时迹应用后，点击 **安装** 按钮
4. 等待下载和安装完成

### 方式二：通过命令行安装

如果您熟悉命令行操作，也可以使用以下命令进行安装：

```bash
appcenter-cli install-fpk dev.xuanran.timetrace.fpk
```

---

## 二、启动应用

安装完成后，在 fnOS 桌面上找到时迹图标，点击即可启动应用。首次启动可能需要几秒钟的初始化时间。

---

## 三、授权数据目录

时迹作为独立应用运行在 fnOS 的应用沙箱中，默认仅能访问自身的应用目录，无法直接读取 NAS 上的其他文件夹。因此，在首次使用扫描或归档功能之前，需要先将待操作的目录授权给时迹。

### 操作步骤

1. 打开 **应用中心 → 时迹 → 应用设置 → 授权目录**
   > 此入口仅管理员账号可见。
2. 在目录选择器中，勾选需要让时迹访问的**源目录**（待扫描的文件夹）和**目标目录**（归档存放位置），完成授权。
3. 返回时迹应用，在新建扫描任务时，目录选择器将自动列出已授权的目录供快捷选择。

### 权限说明

- **管理员账号**：可以看到并授权所有目录。
- **普通用户账号**：可选目录范围会自动排除其他用户的个人空间（即 `/vol{n}/{uid}/...` 形式的路径），确保不同用户之间的数据互相隔离。
- **团队共享空间**：不受用户隔离限制，所有已授权用户均可访问。

---

## 四、创建扫描任务

### 4.1 进入任务管理

启动时迹后，在主界面点击 **任务管理** 页面。

### 4.2 新建扫描任务

点击 **新建扫描任务** 按钮，填写以下信息：

| 配置项 | 说明 |
|--------|------|
| **任务名称** | 为任务指定一个易于识别的名称 |
| **源目录** | 需要扫描的文件夹路径（从已授权目录中选择） |
| **目标目录** | 归档文件的存放位置 |
| **归档模式** | **复制**：保留源文件不动，将副本归档到目标目录；**移动**：将源文件移动到目标目录，源位置不再保留 |

### 4.3 开始扫描

确认配置无误后，点击 **开始扫描**。扫描过程中可以实时查看进度和已发现的文件列表。

---

## 五、执行归档

扫描完成后，系统会展示待归档文件的完整列表。请仔细检查文件列表，确认无误后点击 **开始归档**。

> **注意**：如果选择了「移动」模式，归档完成后源文件将被删除。建议首次使用时选择「复制」模式，确认归档结果正确后再切换为「移动」模式。

---

## 六、功能详解

### 6.1 智能扫描

时迹会递归扫描源目录下的所有文件（不区分文件类型），并在扫描过程中执行以下操作：

1. **提取时间信息**：按优先级依次尝试以下方式获取文件的拍摄/创建时间：
   - 读取 EXIF 拍摄日期（适用于 JPEG、TIFF、HEIC 等格式）
   - 识别文件名中的时间戳（支持 13 位 Unix 毫秒格式）
   - 使用文件系统修改时间作为兜底方案

2. **计算 SHA1 哈希**：为每个文件生成 SHA1 摘要值，用于后续的去重和重复文件查找。

3. **增量扫描**：已处理过的文件会被自动跳过，仅处理新增和发生变更的文件，大幅提升后续扫描效率。

### 6.2 分类规则

分类规则决定了文件将被归档到哪个子文件夹。系统支持以下五种匹配方式：

| 匹配方式 | 说明 | 示例 |
|---------|------|------|
| 扩展名 | 按文件扩展名匹配 | `.jpg`、`.mp4`、`.pdf` |
| 文件名 | 按文件名模式匹配（支持通配符 `*`） | `*微信*`、`IMG_*`、`DSC_*` |
| 源文件夹 | 按源文件所在目录路径匹配 | `/DCIM/`、`/WeChat/`、`/Download/` |
| EXIF 品牌 | 按拍摄设备品牌匹配 | `Apple`、`Canon`、`Samsung` |
| EXIF 型号 | 按拍摄设备型号匹配 | `iPhone 15 Pro`、`Canon EOS R5` |

规则支持**优先级排序**，排在前面的规则优先匹配。当文件同时满足多条规则时，仅应用优先级最高的规则。

### 6.3 命名规则

命名规则支持模板化重命名，通过变量占位符自定义归档后的文件名格式：

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `{date}` | 文件日期（格式：YYYY-MM-DD） | `2024-06-15` |
| `{year}` | 年份 | `2024` |
| `{month}` | 月份 | `06` |
| `{day}` | 日期 | `15` |
| `{sha1}` | 文件 SHA1 哈希（取前 8 位） | `a1b2c3d4` |
| `{ext}` | 文件扩展名（含点号） | `.jpg` |
| `{name}` | 原始文件名（不含扩展名） | `IMG_001` |

**命名示例**：

- `{date}_{sha1}{ext}` → `2024-06-15_a1b2c3d4.jpg`
- `{year}/{month}/{name}{ext}` → `2024/06/IMG_001.jpg`

### 6.4 定时任务

时迹支持基于 Cron 表达式的定时调度，可配置系统在指定时间自动执行扫描和归档任务，无需手动操作。

**Cron 表达式格式**：

```
分 时 日 月 周
```

**常用表达式示例**：

| 表达式 | 执行周期 |
|--------|---------|
| `0 2 * * *` | 每天凌晨 02:00 执行 |
| `0 */6 * * *` | 每 6 小时执行一次 |
| `0 3 * * 1` | 每周一凌晨 03:00 执行 |
| `0 0 1 * *` | 每月 1 日零点执行 |

### 6.5 重复文件查找

基于 SHA1 哈希值查找目录中的重复文件。查找结果会按哈希值分组展示，方便您逐组对比和清理，释放存储空间。

## 七、归档目录结构

归档完成后，文件将按以下结构自动组织：

```
目标目录/
├── 2024/
│   ├── 01/
│   │   ├── Photos/        # 图片文件
│   │   ├── Videos/        # 视频文件
│   │   ├── Music/         # 音频文件
│   │   ├── Documents/     # 文档文件
│   │   ├── Archives/      # 压缩包文件
│   │   └── Others/        # 其他类型文件
│   ├── 02/
│   │   └── ...
│   └── ...
└── 2025/
    └── ...
```

每个年份目录下按月份分子目录，每个月份目录内按文件类型归类。此结构便于按时间维度进行备份和检索。
