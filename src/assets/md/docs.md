# 使用文档

**时迹（TimeTrace）** 旨在帮助用户将长期散落在不同目录、不同设备中的照片、视频等文件，按照统一的时间规则进行自动归档与整理，构建清晰、稳定、可持续维护的数字资料库。

其通过智能扫描与多源时间信息提取能力，自动识别文件时间，并按照 `{年}/{月}/{分类}/` 的结构进行归档。相比传统按设备、应用或来源混乱存放的方式，时迹具备天然的增量特性，新增文件通常只会进入对应月份目录，无需重复整理历史数据，使数据结构更加直观、统一且长期可维护。

> 因飞牛应用商店审核巨慢，因此部分文档内容可能与应用商店下载的版本存在区别，以最新版本为主。
---

## 核心功能

时迹围绕"自动识别、自动归档、长期可维护"这一目标，提供以下核心能力：

| 功能 | 说明                                             |
|------|------------------------------------------------|
| 智能扫描 | 递归遍历源目录下的全部文件，自动提取时间信息并计算文件哈希值                 |
| 增量识别 | 基于文件大小与修改时间判断文件是否发生变化，已处理且未变化的文件自动跳过           |
| 自动归档 | 按 `年/月/分类` 三级结构将文件复制或移动至目标目录                   |
| 内置去重 | 扫描阶段即基于 SHA1 哈希值识别内容重复的文件，重复文件不会被重复归档          |
| 分类规则 | 支持按扩展名、文件名、来源文件夹、EXIF 品牌与型号等维度，将文件归入自定义子目录     |
| 自动命名 | 基于条件与模板变量，为满足条件的文件生成统一格式的归档文件名                 |
| 定时调度 | 基于 Cron 表达式自动执行扫描与归档，可对多个不同的目录分别配置             |
| 媒体修复工具 | 查找归档库中缺少 EXIF 拍摄时间的文件，并批量将归档时推算的时间写回文件元数据 |
| 重复文件查找 | 独立工具，针对任意目录查找内容相同的重复文件并清理                 |
| 加密冷备份 | 独立工具，将指定目录整体打包为加密压缩包，用于导出到外部存储的灾备场景            |
| 仪表盘统计 | 按类型、状态、年月等维度汇总当前账户下的文件处理情况                     |
| 操作日志 | 完整记录扫描、归档、删除等操作的执行结果，便于追溯                      |
| 多用户隔离 | 不同用户的任务、文件记录、定时任务、日志严格隔离，互不可见                  |
| 数据库备份 | 管理员可将全机数据导出为 SQLite 文件，或导入以整体还原           |

## 授权数据目录(重要)

出于安全考虑，时迹运行于 fnOS 的应用沙箱环境中，**默认仅能访问自身的应用目录。若需对 NAS 上的其他目录执行扫描或归档操作，须先由管理员完成目录授权。**

**操作路径**：应用中心 → 时迹 → 应用设置 → 授权目录（该入口仅对管理员账号可见）。

目录授权是**应用级别**的设置：管理员授权一次后，理论上所有登录用户都能通过时迹读取 & 操作该目录，但时迹会在此基础上叠加一层个人空间保护，以防止用户能够访问其他人的数据，从而达到数据隔离：

- 形如 `vol{n}/{用户 ID}/...` 的**个人空间**路径，即使已被整体授权，也只对该用户 ID 对应的账号可见和可用，其他用户在目录选择器和实际扫描/归档时都无法访问；
- 形如 `vol{n}/@团队名/...` 的**团队共享空间**，以及其他无法识别归属的路径，不受个人空间限制，授权后所有用户均可使用。

管理员账号在权限范围上**不享有特殊待遇**：管理员同样只能扫描和归档自己有权访问的目录，也只能查看自己创建的任务、文件记录与日志（详见下文[多用户与数据隔离](#多用户与数据隔离)）。
> 例如：管理员对时迹授权了`vol2/1002/Photos`的文件夹，其中`vol2`的`2`为`存储空间2`，`1002`为用户ID，又因该目录为该用户的私有数据目录。
> 因此为保证数据隔离，除`1002`该用户，其他任意用户（包括管理员）均无法对此目录进行任何读写操作。
> 
> * 时迹开发该隔离限制的原因在于非管理员用户无法对时迹授予其用户目录的操作权限；因此任何目录都必须由管理员手动授权时迹后，非管理员用户才可在自己的用户空间中使用时迹。

## 使用流程

时迹的全部操作围绕**任务**展开，整体流程可概括为"新建任务 → 扫描 → 归档"三个阶段，定时任务则是将这一整体流程自动化、周期化执行的方式。

### 第一步：新建任务

完成目录授权后，进入 **任务管理** 页面，点击 **新建扫描任务**，依次完成以下配置：

| 配置项 | 说明 |
|--------|------|
| 源目录 | 需要扫描的文件夹，须从已授权目录范围中选择 |
| 目标目录 | 归档后文件的存放位置，同样须在授权范围内 |
| 归档模式 | **复制**：保留源文件，在目标目录生成副本；**移动**：归档后删除源文件 |
| 扫描后自动归档 | 开启（默认）：扫描完成后自动归档所有待处理文件；关闭：仅扫描，归档需后续在「文件浏览」中手动触发 |

为避免因配置不当导致数据丢失，建议在首次使用时选择"复制"模式，待确认归档结果符合预期后，再根据实际需要切换为"移动"模式。

### 第二步：扫描

点击确认后，时迹将对源目录执行递归扫描，逐一提取每个文件的时间信息并计算其哈希值（具体机制见下文[智能扫描机制](#智能扫描机制)）。扫描过程中产生的记录会实时写入数据库，并可在 **文件浏览** 页面查看——该页面汇总了当前账户下时迹处理过的所有文件及其状态，便于随时核对扫描范围与处理进度。

任务列表会同时展示本次扫描中新增、变化、未变化、消失（源文件已被删除）的文件数量，以及成功、跳过、出错的统计，方便快速判断扫描结果是否符合预期。

### 第三步：归档

当本批次扫描全部完成后：

- 若任务已启用「扫描后自动归档」，时迹会依据扫描阶段得出的时间、分类、命名信息，按所配置的归档模式（复制或移动），将本批次产生的待处理文件自动整理至目标目录下对应的 `年/月/分类` 路径，并自动创建一个关联的归档任务用于展示进度；
- 若未启用自动归档，本批次文件将保持「待处理」状态，可在 **文件浏览** 页面按状态筛选出这些文件，勾选后点击「批量归档」手动触发。

扫描和归档在系统中分别是独立的任务记录，可在 **任务管理** 页面统一查看；同一时刻，全局只允许一个扫描任务和一个归档任务运行，若此时其他用户新建任务会提示存在任务正在运行，不会并行抢占。

若归档任务因应用重启等原因被中断，重新打开应用后该任务会自动转为「已中断」状态，可在任务列表中点击「重试」从断点继续归档；扫描任务不支持断点续传，中断后需重新创建。

### 定时任务：流程自动化

定时任务的本质，是将上述"扫描 → 归档"整套流程按 Cron 表达式设定的周期自动重复执行，使新增文件能够持续、无人值守地被归档，无需用户每次手动新建或触发任务（配置方式见下文[定时调度](#定时调度)）。

## 文件浏览

文件浏览页面汇总了当前账户下时迹处理过的全部文件记录，覆盖所有任务、所有扫描批次。支持按状态（待处理 / 已归档 / 已跳过 / 错误）、文件类型（照片 / 视频 / 音频 / 文档 / 其他）、年份、月份及关键词进行筛选检索。

点击单条记录可查看文件详情，包括原始文件名与格式化后的文件名、源路径与目标路径、文件大小、SHA1、MIME 类型、EXIF 日期/品牌/型号、文件名日期、修改时间、最终采用的归档年月，以及出错或跳过的具体原因。

该页面同时提供以下批量操作：

- **批量归档**：勾选若干「待处理」记录后手动触发归档，适用于未启用自动归档的任务；
- **批量删除**：删除选中的文件记录（仅删除时迹中的记录，不会删除磁盘上的源文件或已归档文件）；
- **按状态清空**：一次性清空某一状态下的全部记录，用于批量清理「已跳过」「错误」等历史记录。

## 智能扫描机制

时迹在扫描过程中，会对源目录下的每一个文件依次执行变化检测、去重判定、时间信息提取与分类命名。

### 增量识别

时迹会为每个源目录维护一份文件索引，记录每个文件的大小与修改时间。扫描时：

- 若文件的大小与修改时间均与索引记录一致，判定为**未变化**，直接跳过，不重新计算哈希或重新提取时间；
- 若文件为新出现或大小/修改时间发生变化，判定为**新增**或**变化**，重新进行后续处理；
- 扫描完成后，索引中存在但本次未被扫描到的文件，会被标记为**消失**（通常意味着源文件已被删除或移动）。

这一机制使得首次扫描完成后，后续的增量扫描只需处理真正发生变化的文件，效率不会随数据总量增长而下降。

### 内容去重

对每个新增或变化的文件，时迹会计算其 SHA1 哈希值，并与已记录的「已归档」「已跳过」「待处理」文件比对。若发现内容完全相同的文件已存在，该文件会被直接标记为**已跳过**，并记录所重复的具体记录编号，不会再次生成归档副本。这一去重判定在扫描阶段自动完成，无需用户额外操作。

### 时间信息提取

时间信息提取遵循以下优先级，逐级尝试，直至成功获取有效时间：

1. **EXIF / 元数据拍摄时间**：图片格式（JPEG、TIFF、HEIC 等）读取 EXIF 标签，视频格式（MP4、MOV、MKV 等）通过 ffprobe 读取容器元数据中的创建时间；
2. **文件名时间戳**：识别文件名中包含的时间信息，支持 13 位 Unix 毫秒时间戳（如微信导出文件名）、`IMG_/VID_` 前缀日期、`Screenshot_` 截图命名、中文前缀日期、连续数字日期等多种常见格式；
3. **文件系统修改时间**：在前述两种方式均无法获取有效时间时，作为兜底依据。

若文件名本身是一段哈希值或随机字符串（不含真实日期信息），时迹会自动识别并跳过文件名时间提取，避免将随机数字误判为时间戳。

首次扫描包含数万级以上文件的目录时，由于需要逐一计算时间信息与哈希值，耗时较长属于正常现象；完成首次扫描后，后续的增量扫描速度将显著提升。

## 分类规则

分类规则用于决定文件最终归档到哪个子文件夹。每个用户拥有自己独立的规则集，首次访问「分类规则」页面时会自动初始化一套内置默认规则，用户可在此基础上自由增删改。

每条规则包含以下要素：

| 要素 | 可选值 |
|------|--------|
| 匹配字段 | 文件扩展名 / 文件名 / 来源文件夹名称 / EXIF 相机品牌（Make）/ EXIF 相机型号（Model） |
| 匹配类型 | 包含 / 开头是 / 结尾是 / 完全匹配 / 正则表达式（`contains` 模式下不区分大小写） |
| 归档到文件夹 | 文件最终归入的子文件夹名称，最终路径为 `{年}/{月}/{此文件夹名}/` |
| 优先级 | 数字越大优先级越高；相同优先级按创建顺序排列 |

规则按优先级从高到低依次匹配，**首条匹配命中即生效**，不再继续匹配后续规则。例如配置「文件名包含 `Screen` → 归档到 `Screenshots`」，则文件名含有 `Screen` 的文件会被优先归入 `Screenshots` 目录，而不再进入默认分类。

若文件未匹配任何已启用的规则，将按扩展名归入以下默认目录：

| 文件类型 | 默认目录 |
|---------|---------|
| 照片 | `Photos` |
| 视频 | `Videos` |
| 音频 | `Audio` |
| 文档 | `Doc` |
| 其他（未识别类型） | `Others` |

所有用户均可在「分类规则」页面自行管理自己的规则，包括新增、编辑、删除与启停操作。各用户的规则相互独立，互不影响（自v1.1.0版本开始）。

## 自动命名

自动命名规则用于自定义归档后的文件名格式。与[分类规则](#分类规则)类似，每个用户拥有独立的命名规则集，按优先级匹配，且可附加生效条件：

- **命名条件**：可留空（对所有文件生效），也可指定 `prefix:前缀`、`suffix:后缀`、`folder:来源文件夹名称包含`、`hash`（文件名形似哈希或随机字符串）、`regex:正则表达式` 中的一种或多种，多个条件用逗号分隔，满足任一条件即生效；
- **命名模板**：仅在条件满足时按模板重命名文件，不满足任何已配置规则的文件将保留原始文件名。

模板中可使用以下变量：

| 变量 | 含义 | 示例值 |
|------|------|--------|
| `{date}` | 日期时间 | `20240115_123456` |
| `{date_short}` | 日期 | `20240115` |
| `{year}` | 年份 | `2024` |
| `{month}` | 月份 | `01` |
| `{day}` | 日 | `15` |
| `{time}` | 时间 | `123456` |
| `{sha1}` | SHA1 哈希前 6 位 | `a3f2b1` |
| `{sha1_full}` | 完整 SHA1 哈希 | `a3f2b1c4...` |
| `{ext}` | 文件扩展名 | `.jpg` |
| `{orig}` | 原始文件名（不含扩展名） | `IMG_001` |
| `{folder}` | 来源文件夹名 | `Camera` |
| `{make}` | EXIF 相机品牌 | `Apple` |
| `{model}` | EXIF 相机型号 | `iPhone 15 Pro` |
| `{counter}` | 自增计数器 | `1` |

例如，配置条件 `prefix:mmexport,hash`、模板 `{date}_{sha1}{ext}`，可将微信导出的、以及文件名形似哈希的文件统一重命名为 `20240115_123456_a3f2b1.jpg` 这类规范格式，而其他正常命名的文件则保持原名不变。

所有用户均可在「自动命名」页面自行管理自己的命名规则，包括新增、编辑、删除与启停操作。各用户的命名规则相互独立，互不影响（自v1.1.0版本开始）。

## 定时调度

时迹支持基于 Cron 表达式为指定的源目录 / 目标目录组合配置定时任务，使「扫描 → 归档」的完整流程能够在指定时间自动重复执行，无需人工触发。每个用户可创建并管理自己的定时任务，相互独立。

每个定时任务可单独配置归档模式（复制/移动）与是否自动归档，也支持点击「立即执行」手动触发一次，便于在等待下次调度前提前验证配置是否正确。常用表达式示例如下：

| 表达式 | 调度周期 |
|--------|---------|
| `0 2 * * *` | 每日 02:00 执行一次 |
| `0 */6 * * *` | 每 6 小时执行一次 |
| `0 3 * * 1` | 每周一 03:00 执行一次 |
| `0 0 1 * *` | 每月 1 日 00:00 执行一次 |

定时任务与手动创建的任务共用同一套全局并发控制：若触发时已有扫描或归档任务在运行，本次定时任务会进行跳过，不会与正在进行的任务冲突。

## 重复文件查找

重复文件查找是一项独立于应用体系之外的免费工具，其可以对指定目录下的文件进行分析，找出其完全相同的文件并按照一定策略进行删除。

使用时，输入一个或多个待检查的目录路径，时迹会采用两阶段策略查找重复文件：先按文件大小分组，仅对大小相同的文件计算 SHA1 哈希值，避免对全部文件逐一计算带来的性能开销。结果按哈希值分组展示，同一组内的文件内容完全相同；每组中**修改时间最早**的文件会默认标记为「保留」（默认修改时间最早可切换），其余文件可勾选后批量删除。结果页支持按扩展名和文件大小范围筛选。

## 多用户与数据隔离

时迹对任务、文件记录、定时任务、操作日志均按账号严格隔离：每个用户只能查看和操作自己创建的数据，管理员在这方面不享有额外权限——既看不到其他用户的任务和文件记录，也不能代替其他用户操作。

以下操作属于全机级别的管理功能，仅限管理员执行：

- **系统设置**中的数据库导出 / 导入，覆盖本机全部用户的扫描归档任务、文件记录、定时任务、分类与命名规则、操作日志等全部数据，用于整机级别的备份、迁移或灾难恢复。导入会整体替换当前数据库内容，执行前请确保没有任务正在运行，并建议先导出当前数据库作为备份。
- **应用日志与运行日志**的下载，入口位于系统设置页，仅管理员可见。

## 会员与使用额度

时迹采用「免费额度 + 终身会员」的授权模式：

- 未开通会员时，单台设备累计最多可创建 **15 次**扫描与归档任务（两者合并计数；若一次操作同时触发了扫描和自动归档，会计为 2 次）。该额度为整机共享，设备上的所有用户共用同一份额度；
- 开通终身会员后不再受次数限制。会员资格与设备的机器码绑定，机器码不变则授权永久有效，卸载重装应用后无需重新购买，可直接通过机器码找回授权；
- 授权校验在本地离线完成（基于签名校验，无需联网），仅在购买、查询价格或找回授权时才会短暂连接开发者的授权服务器，且通信内容不包含任何用户个人信息；
- 开通与激活会员仅限管理员账号操作。

## 归档目录结构

归档完成后，文件将按以下结构自动组织：

```txt
目标目录/
├── 2024/
│   ├── 01/
│   │   ├── Photos/
│   │   ├── Videos/
│   │   ├── Audio/
│   │   ├── Doc/
│   │   └── Others/
│   ├── 02/
│   └── ...
└── 2025/
    └── ...
```

文件按"年份 → 月份 → 文件类型（或自定义分类规则所指定的文件夹）"逐级组织，结构在不同年份、不同月份之间保持完全一致，便于用户按时间维度进行备份、检索与长期归档管理。
