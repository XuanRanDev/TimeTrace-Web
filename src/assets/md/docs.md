# 使用教程

本指南旨在帮助用户全面了解时迹的功能特性，并指导用户完成从安装到日常使用的完整流程。

---

## 功能概览

时迹自动提取文件的拍摄时间或创建时间，将散落各处的文件按 `年/月/分类` 的结构统一归档。这种基于时间的归档方式天然适配增量备份，每月新增文件集中在固定目录，备份时只需同步当月目录，无需全量扫描。同时，按时间组织的文件结构便于分批次上传至多个云存储平台进行异地备份，避免因文件散乱导致的重复上传和存储空间浪费。

---

## 一、安装与启动

### 1.1 通过应用中心安装

1. 登录 fnOS 系统，打开 **应用中心**
2. 在搜索栏中输入「**时迹**」
3. 在搜索结果中找到时迹应用，点击 **安装**
4. 等待应用下载并安装完成

### 1.2 启动应用

安装完成后，在 fnOS 桌面或应用列表中找到时迹图标，点击启动。首次启动可能需要数秒完成初始化。

---

## 二、授权数据目录

### 2.1 背景说明

时迹运行在 fnOS 的应用沙箱环境中，默认仅具备访问自身应用目录的权限。若需扫描或归档 NAS 上的其他目录，须事先完成目录授权操作。

### 2.2 授权步骤

1. 以管理员身份登录 fnOS
2. 打开 **应用中心 → 时迹 → 应用设置 → 授权目录**
   > 此入口仅对管理员账号可见。
3. 在目录选择器中，勾选需要授权的**源目录**和**目标目录**
4. 确认授权后，返回时迹应用即可在目录选择器中看到已授权的目录

### 2.3 权限说明

| 账号类型 | 可见目录范围 |
|---------|------------|
| 管理员账号 | 所有目录 |
| 普通用户账号 | 仅限已授权的目录，且自动排除其他用户的个人空间（`/vol{n}/{uid}/...`） |

> 团队共享空间不受用户隔离限制，所有已授权用户均可访问。

---

## 三、创建扫描任务

### 3.1 进入任务管理

启动时迹后，在主界面导航至 **任务管理** 页面。

### 3.2 配置任务参数

点击 **新建扫描任务**，根据以下说明填写各项配置：

| 配置项 | 必填 | 说明 |
|--------|------|------|
| 任务名称 | 是 | 为任务指定一个便于识别的名称 |
| 源目录 | 是 | 需要扫描的文件夹路径（从已授权目录中选择） |
| 目标目录 | 是 | 归档文件的存放位置 |
| 归档模式 | 是 | **复制**：保留源文件，在目标目录生成副本；**移动**：将源文件移至目标目录 |

### 3.3 执行扫描

配置完成后，点击 **开始扫描**。系统将递归扫描源目录下的所有文件，扫描过程中可实时查看进度及已发现的文件列表。

---

## 四、执行归档

扫描任务完成后，系统将展示待归档文件的完整列表。请逐一确认文件信息，确认无误后点击 **开始归档**。

> **重要提示**：选择「移动」模式时，归档操作完成后源文件将被删除。建议首次使用时选择「复制」模式，待确认归档结果正确后再切换为「移动」模式，以避免误操作导致数据丢失。

---

## 五、核心功能

### 5.1 智能扫描引擎

时迹采用多层级扫描策略，递归遍历源目录下的所有文件（不区分文件类型），并在扫描过程中执行以下处理：

**时间信息提取**（按优先级排序）：

1. **EXIF 拍摄日期**：从 JPEG、TIFF、HEIC 等支持 EXIF 的图片格式中读取拍摄时间
2. **文件名时间戳**：识别文件名中嵌入的 13 位 Unix 毫秒时间戳
3. **文件系统修改时间**：作为兜底方案，使用文件的最后修改时间

**哈希计算**：为每个文件生成 SHA1 摘要值，用于去重判定和重复文件查找。

**增量扫描机制**：已处理过的文件将被自动跳过，仅对新增和发生变更的文件进行处理，显著提升后续扫描效率。

### 5.2 分类规则引擎

分类规则决定文件的归档目标子文件夹。系统支持以下五种匹配方式：

| 匹配方式 | 匹配对象 | 通配符支持 | 示例 |
|---------|---------|-----------|------|
| 扩展名 | 文件扩展名 | 否 | `.jpg`、`.mp4`、`.pdf` |
| 文件名 | 文件名 | 支持 `*` | `*微信*`、`IMG_*`、`DSC_*` |
| 源文件夹 | 源文件所在目录路径 | 否 | `/DCIM/`、`/WeChat/` |
| EXIF 品牌 | 拍摄设备品牌 | 否 | `Apple`、`Canon`、`Samsung` |
| EXIF 型号 | 拍摄设备型号 | 否 | `iPhone 15 Pro`、`Canon EOS R5` |

规则支持优先级排序，排在前面的规则优先匹配。当文件同时满足多条规则时，仅应用优先级最高的规则。

### 5.3 命名规则模板

命名规则通过变量占位符实现文件的自动化重命名：

| 变量 | 含义 | 格式示例 |
|------|------|---------|
| `{date}` | 文件日期 | `2024-06-15` |
| `{year}` | 年份 | `2024` |
| `{month}` | 月份 | `06` |
| `{day}` | 日期 | `15` |
| `{sha1}` | SHA1 哈希值（前 8 位） | `a1b2c3d4` |
| `{ext}` | 文件扩展名（含点号） | `.jpg` |
| `{name}` | 原始文件名（不含扩展名） | `IMG_001` |

**应用示例**：

- `{date}_{sha1}{ext}` → `2024-06-15_a1b2c3d4.jpg`
- `{year}/{month}/{name}{ext}` → `2024/06/IMG_001.jpg`

### 5.4 定时调度

支持基于 Cron 表达式的定时任务调度，可在指定时间自动执行扫描和归档操作。

**Cron 表达式格式**：

```
分 时 日 月 周
```

**典型应用场景**：

| 表达式 | 调度周期 |
|--------|---------|
| `0 2 * * *` | 每日 02:00 |
| `0 */6 * * *` | 每 6 小时 |
| `0 3 * * 1` | 每周一 03:00 |
| `0 0 1 * *` | 每月 1 日 00:00 |

### 5.5 重复文件检测

基于 SHA1 哈希值对目录中的重复文件进行检测。检测结果按哈希值分组展示，便于用户逐组对比并清理冗余文件。

## 六、归档目录结构

归档完成后，文件将按以下层级结构自动组织：

```
目标目录/
├── 2024/
│   ├── 01/
│   │   ├── Photos/        # 图片文件
│   │   ├── Videos/        # 视频文件
│   │   ├── Music/         # 音频文件
│   │   ├── Documents/     # 文档文件
│   │   ├── Archives/      # 压缩包文件
│   │   └── Others/        # 其他类型文件
│   ├── 02/
│   │   └── ...
│   └── ...
└── 2025/
    └── ...
```

文件按「年份 → 月份 → 文件类型」三级目录组织，便于按时间维度进行备份与检索。
