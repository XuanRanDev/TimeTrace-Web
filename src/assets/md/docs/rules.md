# 分类规则

分类规则决定了文件归档到哪个子文件夹。时迹的归档路径结构是 `{年}/{月}/{分类目录}/`，前两级由文件的拍摄时间决定，最后一级「分类目录」就由这里的规则引擎决定。

理解分类规则之前，有必要先理解它在整个归档体系中的位置：**分类规则只影响第三级目录的名称，不会影响时间的提取结果，也不会改变文件归入哪一年哪一个月**。它解决的是「同年同月的不同类型文件如何进一步细分」这个问题。

每个用户拥有自己独立的规则集，首次访问本页面时会自动初始化一套内置默认规则。您可以在此基础上增删改，各用户之间的规则相互隔离，互不影响。

---

## 规则的匹配逻辑

规则引擎在归档时对每个文件依次进行匹配，核心逻辑如下：

1. 将所有**已启用**的规则按优先级从高到低排序
2. 从最高优先级的规则开始，逐条尝试匹配当前文件
3. **首条命中的规则即生效**，该文件归入这条规则指定的文件夹，不再继续匹配后续规则
4. 若遍历完所有规则均未命中，按文件类型（照片/视频/音频/文档/其他）归入内置默认目录

这意味着规则之间的**顺序（优先级）至关重要**。您需要将更具体、更精确的规则设置为更高优先级，将更宽泛的通配规则设置为更低优先级，否则宽泛规则会「拦截」所有文件，让后续的精确规则永远无法被触发。

---

## 规则结构

每条规则包含以下要素：

**匹配字段**：用哪个文件属性来做判断：

| 字段 | 说明 | 使用场景 |
|------|------|---------|
| 文件扩展名 | 如 `.jpg`、`.mp4` | 将特定格式的文件归类，例如将 `.raw` 归入 `RAW` 目录 |
| 文件名 | 文件的完整原始文件名（含扩展名） | 按命名模式分类，例如名称包含 `Screenshot` 的归入截图目录 |
| 来源文件夹名称 | 文件所在目录的**直接父文件夹**名称 | 按来源设备或相册分类，例如 `DCIM` 目录的文件归入 `Camera` |
| EXIF 相机品牌（Make） | EXIF 中的相机厂商信息 | 区分不同品牌设备拍摄的照片，例如 `Apple` → `iPhone` 目录 |
| EXIF 相机型号（Model） | EXIF 中的具体型号 | 更细粒度地区分设备，例如 `iPhone 15 Pro` 单独归类 |

**匹配类型**：关键词与字段值的比较方式：

| 类型 | 说明 | 注意 |
|------|------|------|
| 包含（contains） | 字段值中包含关键词即匹配 | 不区分大小写；最宽泛，容易误匹配 |
| 开头是（starts_with） | 字段值以关键词开头 | 区分大小写 |
| 结尾是（ends_with） | 字段值以关键词结尾 | 常用于扩展名匹配，如 `.jpg` |
| 完全匹配（equals） | 字段值与关键词完全一致 | 最精确，不会误匹配 |
| 正则表达式（regex） | 使用正则表达式匹配 | 最灵活，适合复杂的命名模式 |

**归档到文件夹**：命中此规则的文件将进入 `{年}/{月}/{此名称}/` 路径。文件夹名称可以自由定义，建议使用简洁的英文或拼音，避免空格与特殊字符，以确保跨操作系统兼容性。

**优先级**：数字越大越先匹配。建议为不同层次的规则划分明显的优先级区间，便于管理，例如：200 以上给精确规则，100–200 给一般规则，100 以下给兜底规则。

## 内置默认分类

若文件未命中任何已启用的规则，时迹根据文件 MIME 类型将其归入以下内置默认目录：

| 文件类型 | 默认目录 |
|---------|---------|
| 照片（JPEG、PNG、HEIC 等） | `Photos` |
| 视频（MP4、MOV、MKV 等） | `Videos` |
| 音频（MP3、FLAC 等） | `Audio` |
| 文档（PDF、DOCX 等） | `Doc` |
| 其他（未识别 MIME 类型） | `Others` |

默认分类是兜底机制，始终生效，无法被删除或关闭。

## 配置建议：从具体到宽泛

以下是一个典型的规则配置场景示例，展示了如何利用优先级正确地组织规则：

**目标**：将截图归到 `Screenshots`，将 RAW 格式照片归到 `RAW`，将其余照片和视频分别归到 `Photos` 和 `Videos`。

| 优先级 | 字段 | 类型 | 关键词 | 归档到 |
|--------|------|------|--------|--------|
| 300 | 文件名 | 包含 | `screenshot` | `Screenshots` |
| 200 | 文件扩展名 | 结尾是 | `.arw` | `RAW` |
| 200 | 文件扩展名 | 结尾是 | `.cr3` | `RAW` |
| 100 | 文件扩展名 | 结尾是 | `.jpg` | `Photos` |
| 100 | 文件扩展名 | 结尾是 | `.mp4` | `Videos` |

若截图规则优先级低于扩展名规则，那么名为 `Screenshot_20240101.jpg` 的截图会先被 `.jpg → Photos` 规则命中，进入 `Photos` 目录，而无法到达截图规则，导致分类失效。

---

## 规则只在归档阶段生效

规则的修改不会对已归档的文件产生追溯影响。您在本页面保存规则后，新的规则将在**下一次归档任务**中生效，已完成归档的历史文件的存放路径不会自动改变。若希望按新规则重新整理历史文件，需要重新执行扫描和归档操作。
