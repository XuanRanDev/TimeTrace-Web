# 媒体修复工具

媒体修复工具解决一个特定场景下的问题：**文件已经被时迹归档到了正确的年月目录，但文件本身的 EXIF 元数据中缺少拍摄时间，导致在手机相册、图片管理软件或其他按 EXIF 日期排序的工具中，这些照片无法被正确排序或归类。**

---

## 为什么会出现「没有 EXIF 日期」的照片

EXIF（Exchangeable Image File Format）是嵌入在图片文件内部的元数据标准，其中包含相机品牌、型号、拍摄时间、GPS 坐标等信息。相机、手机拍摄的原始照片通常自带完整的 EXIF 数据，但以下情况会导致 EXIF 拍摄时间缺失：

- **截图**：截图工具生成的图片通常不包含 EXIF 拍摄时间，因为截图的「拍摄时刻」没有标准化写入 EXIF 的机制
- **聊天软件导出的图片**：微信、QQ 等软件导出的图片，部分来源于网络转发，原始 EXIF 已被平台清除以保护隐私
- **下载的图片**：来自网页、社交媒体的图片，元数据通常在发布时已被压缩服务器剥离
- **格式转换**：经过格式转换或编辑软件处理的文件，EXIF 可能在转换过程中丢失

例如飞牛相册在给照片定位时间时，会优先取其照片内部元数据的时间，并以此在时间轴当中显示；但当照片不存在元数据或元数据已丢失时，飞牛相册会根据文件的创建、修改时间作为时间轴的依据，但我们日常的某些操作，例如从网盘下载下来、文件由外部上传、文件移动等会导致其创建、修改时间被改写，进而导致飞牛相册某些图片出现在错误的时间轴位置。

时迹的媒体修复工具，则是专门解决这种情况。

---

## 工具的工作原理

媒体修复工具会读取时迹数据库中每个已归档文件的「最终日期」，并将这个日期**写回**到磁盘文件的 EXIF 元数据中，具体写入以下标签：

- `DateTimeOriginal`（拍摄时间）
- `CreateDate`（创建时间）
- `ModifyDate`（修改时间）

写入完成后，这些文件在 iOS 相册、macOS Photos、Windows 照片、Lightroom 等任何遵循 EXIF 标准的软件中，都能被正确识别为对应时间拍摄的照片，时间线排序恢复正常。

**重要前提**：「最终日期」必须是准确的。如果时迹推算出的日期本身有误，将这个错误日期写入 EXIF 反而会引入新的问题。因此，**建议先在操作 写回EXIF文件 之前核对「最终日期」是否正确**，如有偏差，先通过「修正归档日期」功能更正，再执行 EXIF 写回。

---

## 筛选待处理文件

页面默认开启「**仅显示无 EXIF 文件**」筛选，只列出 `exif_date` 字段为空的已归档文件——这些正是需要补全 EXIF 日期的候选文件。

您还可以结合以下筛选条件缩小范围：

- **文件类型**：通常只需处理「照片」类型，视频格式目前不支持 EXIF 写回
- **年份 / 月份**：按归档时间分批处理，避免一次性操作量过大
- **关键词**：搜索特定文件名，定位某一来源的文件

---

## 支持写回的文件格式

| 支持 | 不支持 |
|------|--------|
| `.jpg` / `.jpeg` | `.mp4`、`.mov`、`.mkv`（视频） |
| `.tiff` / `.tif` | `.png`（PNG 不使用 EXIF 标准） |
| `.heic` / `.heif` | `.gif`、`.webp`、`.bmp` |

不支持的格式在处理结果中会标记为「不支持」并自动跳过，不影响其他文件的写回。
> 暂不支持其他格式写回的原因在于：需要引入外部工具支持，会显著造成安装包体积增加，若后续该功能被大量使用将再排期开发。

---

## 执行写回与结果说明

勾选目标文件后，点击工具栏的**写回 EXIF** 按钮。时迹处理完成后会弹出结果统计：

| 状态 | 含义 | 处理建议 |
|------|------|---------|
| 已写入 | 成功将日期写入 EXIF | 无需操作 |
| 已跳过 | 文件已有 EXIF 拍摄时间，未做修改 | 无需操作，原有 EXIF 保留 |
| 不支持 | 文件格式不在支持列表中 | 无法处理，跳过即可 |
| 文件不存在 | 归档路径对应的磁盘文件已不存在 | 检查文件是否被手动删除或移动 |
| 失败 | 写入过程中发生其他错误 | 检查目标文件的磁盘权限 |

写回成功后，时迹数据库中该文件记录的 `exif_date` 字段会同步更新，「仅显示无 EXIF 文件」筛选会即时反映最新状态——该文件将从列表中消失，说明它已不再需要处理。

---

## 注意事项

**写回操作会直接修改磁盘文件**，虽然只修改文件头部的元数据而不影响图像内容，但从文件操作层面看属于不可撤销的写入。执行前请确认：

1. 文件浏览中该文件的「最终日期」确实准确
2. 如有顾虑，可先对少量文件试写回，在图片软件中验证效果后再批量处理

**单次最多处理 500 个文件**，超出部分请分批操作。

**写回操作不会记录到操作日志**，而是记录在应用日志当中。
