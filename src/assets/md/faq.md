# 常见问题

本页汇总了用户在使用时迹过程中较为常见的疑问，按主题分类列出，供使用前参考查阅。

---

## 权限与授权

### 提示「目录未授权」或「无权限访问该目录」

该提示表明目标目录尚未获得时迹的访问权限。时迹运行于 fnOS 的应用沙箱中，默认仅能访问自身的应用目录，对其他目录的扫描与归档操作均须事先完成授权。

请以管理员身份登录 fnOS，进入 **应用中心 → 时迹 → 应用设置 → 授权目录**，勾选需要授权的目录后，返回时迹应用并刷新页面即可生效，详情请查看[授权数据目录](/docs#授权数据目录(重要))。

### 归档后的文件在重复查找等功能中提示「无法读取」或「权限不足」

这是 fnOS 1.2.00x 版本升级 Windows ACL 功能后可能出现的问题。升级后，由旧版时迹写入的归档文件可能带有源文件的权限信息，导致应用进程无法重新读取这些文件，即使文件所有者显示正确。

**v1.2.1 已修复此问题**，新版本归档的文件会正确继承目标目录的权限，不再出现此情况。

**已归档的旧文件需手动修复**：进入 fnOS 文件管理器，找到您的归档总目录，依次点击「详细信息 → 权限 → 高级 → 重置子级权限」，让所有子文件重新继承目录权限。操作完成后，重复查找等功能即可正常读取这些文件。

### 为什么我授权了某个目录，却无法选择它

该目录可能属于其他用户的个人空间。目录授权是应用级别的设置，管理员授权后所有用户都能在目录选择器中看到已授权的目录，但形如 `vol{n}/{用户 ID}/...` 的个人空间路径只对其归属用户本人可用，以防止跨用户访问私人数据。团队共享空间不受此限制。

## 任务与归档行为

### 为什么我新建的任务一直显示「等待中」，没有立即开始

时迹在全局范围内同一时刻只允许一个扫描任务和一个归档任务运行。若此时已有其他任务（包括其他用户创建的任务，或某个定时任务）正在执行，新任务会自动排队，待当前任务结束后依次执行，不会并行处理多个扫描或归档。

### 文件状态显示「已跳过」是什么原因

最常见的原因是该文件的内容（按 SHA1 哈希值判断）与已经记录过的文件完全相同，时迹在扫描阶段会自动识别这类重复文件并跳过，不再生成归档副本，文件详情中会注明所对应的原始记录编号。

### 任务状态显示「已中断」，应该怎么处理

通常是应用在任务执行期间被重启或异常退出导致的，处理方式取决于任务类型：

- **归档任务**：支持断点续传，点击任务行中的「重试」按钮，时迹会从中断处继续处理，无需从头开始。
- **扫描任务**：不支持断点续传，但中断前已完成识别的文件不会丢失。任务行中会显示「归档遗留」按钮，点击后可对中断前已识别、尚未归档的文件补充执行归档。若需要重新完整扫描，新建一个扫描任务即可，已处理过的文件会被自动跳过。

## 数据隔离与安全

### 时迹如何保证多用户之间的数据隔离

时迹采用严格的数据隔离机制：每个用户仅可查看和操作由本人创建的任务、文件记录、定时任务及操作日志，管理员在这方面不享有额外权限。即便某个个人空间目录被整体授权，该目录下的数据也仅对其所属用户可见，不存在跨用户数据泄露的可能。

唯一的例外是系统设置中的数据库导出导入，该功能覆盖全机所有用户的数据，属于整机级别的管理操作，仅限管理员账号执行，详见[使用文档「多用户与数据隔离」一节](/docs#多用户与数据隔离)。

### 时迹是否会上传或收集用户数据

时迹遵循最小数据收集原则，所有扫描记录、文件索引、操作日志均存储于 fnOS 本地的 SQLite 数据库中，不会上传至任何外部服务器，也不集成任何第三方分析工具、广告 SDK 或追踪组件。更多细节请参见[用户协议与隐私政策](/terms)。

## 使用限制

### 任务创建次数是否有限制

未开通会员的情况下，单台设备累计最多可创建 **15 次**扫描与归档任务，两者合并计数：若一次操作同时触发了扫描和自动归档，会计为 2 次。该额度为整机共享，设备上的所有用户共用同一份额度。开通终身会员后，机器码绑定永久生效，不再受次数限制。详见[使用文档「会员与使用额度」一节](/docs#会员与使用额度)。

### 谁可以修改分类规则和命名规则

每个用户拥有独立的分类规则与命名规则集，所有用户（包括普通用户）均可在对应页面自行新增、编辑、删除及启停自己的规则，用户之间的规则互不影响。首次访问规则页面时，会自动初始化一套内置的默认规则，详见[分类规则](/docs#分类规则)与[自动命名](/docs#自动命名)。

## 性能与文件支持

### 扫描速度受哪些因素影响

扫描速度主要取决于磁盘 I/O 性能与待处理文件数量。首次扫描包含大量文件的目录时，由于需要逐一提取时间信息并计算哈希值，耗时较长属于正常现象；完成首次扫描后，由于已处理文件会被自动跳过，增量扫描速度会显著提升。若源目录所在磁盘当前未被其他高负载任务占用，扫描效率可进一步提高。

### 时迹支持哪些文件格式

时迹对全部文件格式一视同仁，扫描与归档过程不区分文件类型。在时间信息提取方面：图片格式（JPEG、TIFF、HEIC 等）读取 EXIF 元数据，视频格式（MP4、MOV、MKV 等）读取容器元数据中的创建时间；对于无法读取到有效元数据的文件，系统会依次尝试文件名时间戳与文件系统修改时间作为替代依据。

## 反馈与支持

### 如何提交问题反馈或功能建议

欢迎通过以下任一方式联系：

- **GitHub Issues**（推荐）：前往[问题反馈](/feedback)页面获取提交入口；
- **电子邮件**：xuanrandev@qq.com。

提交反馈时，建议附上日志文件以便定位问题。管理员可在「系统设置」页面直接下载以下两份日志：

- **运行日志**（`info.log`）：记录应用启动与运行摘要；
- **应用日志**（`archiver.log`）：记录详细的扫描与归档过程，排查归档问题时优先提供此文件。

也可直接在 NAS 上找到这两个文件，路径为 `/vol{x}/@appdata/dev.xuanran.timetrace/`（其中 `x` 为对应存储空间的序号）。
